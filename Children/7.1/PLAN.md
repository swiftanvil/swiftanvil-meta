# Child 7.1 Plan: Naming Cleanup & Package Consolidation

## Status

Planned.

## Goal

Eliminate naming collisions and inconsistencies across the SwiftAnvil organization.
Align repository names, package names, module names, and executable names.

## Critical Issues to Fix

1. **Duplicate package name:** `swiftanvil-anvil-project` and `swiftanvil-cli` both define `package(name: "iFoundation")`. They cannot coexist as SPM dependencies.
2. **Legacy name:** `iFoundation` is the old project name. All references must become `SwiftAnvilCLI` / `swiftanvil`.
3. **Repoâ†’package misalignment:** Several repos have package names that don't match their repo names.
